PRを作成します。

## フォーマット

- タイトル: [{チケット番号}] {チケットのタイトル}
  - 例: [FOO-1234] ○○画面の実装
- Assignee: 指示した人
- 概要欄: 以下で固定とする
    ```
    ## チケット

    - x

    ## 仕様書・Figma

    - x

    ## やったこと

    - x

    ## やってないこと

    - x

    ## 参考リンク

    - x
    ```
- 指示がない限りDraftとする

## 「やったこと」の書き方

- 「コードの変更でやったこと」だけを書く
- 「テストが通ることを確認した」など、コードの変更によるものでないことは書かない
- ファイル名でなく、何をしたかの説明で書く
  - ⭕ ○○のドキュメントを更新した
  - ❌ `README.md` を更新した

## 「やってないこと」の書き方

- 「やる必要があるのにやってないこと」だけを書く（このPRの範囲外にした残作業など）
- 「やる必要がないこと」は書かない
